*===============================================================*
* テストケース仕様書（PGM01 利息計算・決済処理用）             
*---------------------------------------------------------------*
* プログラムID : PGM01                                         
*---------------------------------------------------------------*
* プログラム概要                                                 
* 本テストケースは、銀行ABCの預金管理システムにおける
* PGM01プログラムの利息計算(FUNCTION-001)および精算処理(FUNCTION-002)
* の正当性を検証するためのテストケースを示す。              
*---------------------------------------------------------------*
* テスト目的                                                    
*---------------------------------------------------------------*
* 1. 正常系ケースで利息計算と精算処理が正しく行われること
* 2. 異常系ケースでエラー処理が正しく行われること（SQLエラー・日付不正など）
*---------------------------------------------------------------*
* 前提条件                                                    
*---------------------------------------------------------------*
* - DB_ACCOUNT_SAVINGS, DB_INTEREST_INFO, DB_ACCOUNT_BALANCE
*   がテスト用データで準備されていること
* - プログラムは単一契約ごとに処理する
*---------------------------------------------------------------*
* テストケース一覧                                            
*---------------------------------------------------------------*
* 1. 正常系テストケース (Normal Case)
*---------------------------------------------------------------*
* ケース1 : 非定期預金 (NON-TERM) の利息計算                  
*   - DB_ACCOUNT_SAVINGS.STATUS = '1'                              
*   - SAVING_TYPE = 'NON-TERM'                                      
*   - 経過日数 = 現在日付 - START_DATE                             
*   - 利息 = MONEY_ROOT × INTEREST_RATE × 経過日数 ÷ 365           
*   - 結果：DB_ACCOUNT_SAVINGS.INTEREST, MONEY が更新される     
*---------------------------------------------------------------*
* ケース2 : 定期預金 (FIXED-03, 90日) 満期到達時                
*   - SAVING_TYPE = 'FIXED-03', 経過日数 >= 契約期間             
*   - 利息 = MONEY_ROOT × INTEREST_RATE × 90 ÷ 365                
*   - 結果：DB_ACCOUNT_SAVINGS.INTEREST, MONEY が更新される     
*---------------------------------------------------------------*
* ケース3 : 定期預金 (FIXED-06, 180日) 満期前                    
*   - SAVING_TYPE = 'FIXED-06', 経過日数 < 契約期間              
*   - 利息 = MONEY_ROOT × 非定期利率 × 経過日数 ÷ 365             
*   - 結果：DB_ACCOUNT_SAVINGS.INTEREST, MONEY が更新される     
*---------------------------------------------------------------*
* ケース4 : 精算処理 (FUNCTION-002)                              
*   - DB_ACCOUNT_SAVINGS.STATUS = '1'                              
*   - 現在日付と契約満期日を比較                                 
*   - 満期到達：利息再計算不要                                    
*   - 前倒し：非定期利率で利息再計算                              
*   - DB_ACCOUNT_BALANCE.BALANCE に MONEY を加算                
*   - DB_ACCOUNT_SAVINGS.STATUS = '9' に更新                     
*---------------------------------------------------------------*
* 2. 異常系テストケース (Abnormal Case)
*---------------------------------------------------------------*
* ケースA : SQLエラー発生                                       
*   - DB_INTEREST_INFO テーブルが存在しない、またはアクセス不可
*   - 結果：HV-ABEND-BREAKPOINT にモジュール名格納            
*          ログ出力                                          
*          STOP RUN                                          
*---------------------------------------------------------------*
* ケースB : 開始日 > 終了日の不正データ                         
*   - DB-START-DATE > DB-END-DATE                                
*   - 経過日数計算時に負の日数が発生                             
*   - 対応：WS-DAYS-ACTUAL を 0 に設定                           
*   - 結果：エラーをログ出力、処理継続可                          
*---------------------------------------------------------------*
* ケースC : 無効ステータスの契約                                 
*   - DB_ACCOUNT_SAVINGS.STATUS ≠ '1'                            
*   - 結果：スキップされ、処理対象外                           
*---------------------------------------------------------------*
* ケースD : 定期預金で契約期間が未来日付                         
*   - 現在日付 < START_DATE                                       
*   - 結果：経過日数 = 0、利息 = 0                               
*---------------------------------------------------------------*
* 備考                                                          
*---------------------------------------------------------------*
* - 全てのケースは個別契約単位でテストする                      
* - 利率は必ず DB から取得すること                              
* - Money = Money_Root + Interest が正しく計算されること       
* - 前倒し精算時は非定期利率を使用する
